拖动图表内容

通过按下鼠标左键并移动鼠标来拖动图表的内容，以将其水平平移。
在您放开图表后不久，所有图表都会出现（这个小延迟是设计使然：它可以加快浏览器的速度，让您专注于正在探索的内容）。
平移图表后，所有图表的自动刷新都会停止。要再次启用它，请双击平移的图表。

连点两下

双击图表可将所有图表重置为其默认的自动刷新状态。

SHIFT +拖动

在按下SHIFT键的同时，在图表内容上按下鼠标左键，移动鼠标选择一个区域，进行放大。其他图表也随之而来。 缩放分两个阶段执行：
  • 已加载的图表内容被放大（低分辨率）
  • 从 netdata 服务器传输新数据，以刷新图表以获取更多细节。
图表放大后，所有图表的自动刷新都会停止。 要再次启用它，请双击放大的图表。

反白时间轴

在按下ALT 键的同时，在图表的内容上按下鼠标左键并移动鼠标以选择一个区域。所选的将在所有图表上突出显示。

SHIFT + 鼠标滚轮

当按下 SHIFT 键且鼠标指针位于图表内容上方时，滚动鼠标滚轮以放大或缩小。这种缩放与鼠标指针下方的中心对齐。其他图表也将随之而来。
图表放大后，所有图表的自动刷新都会停止。要再次启用它，请双击放大的图表。

这里是浏览器的设定，每个浏览者都可以拥有自己的设定。这里的设定不会影响 netdata 本身的运作。
设定会立即生效，并且储存在您浏览器的本地储存中 (除了重新整理的 在焦点上/总是 这两个选项以外)。
若要将所有选项重置回预设值 (包括图表大小)，请点选 这里。
  • 效能
  • 同步
  • 显示

何时要重新整理图表？
当设置为在焦点上时，如果页面/标签没有用户的焦点，图表将停止更新。 当设置为总是时，图表将始终刷新。 将其设置为 在焦点上 以在此页面没有您的焦点时降低浏览器的 CPU 要求（并延长笔记本电脑和平板电脑的电池）。 设置为 总是 以在另一个窗口上工作（即更改某些设置）并使图表在此窗口中自动刷新。
要显示那些图表维度？
当设置为非零时，所有值（在当前视图内）都设置为零的维度将不会从netdata服务器传输（除非图表的所有维度都为零，在这种情况下 此设置没有任何作用 - 所有尺寸都被传输和显示）。 当设置为 All 时，将始终显示所有尺寸。 将其设置为非零以降低netdata和浏览器之间传输的数据，降低浏览器对CPU的要求（绘制更少的线）并增加对图例的关注（图例中的条目更少）。
如何处理隐藏图表？
当设置为消去时，不在浏览器当前视口中的图表（位于页面可见区域的上方或下方），将在它们变得可见时被销毁并重新创建 再次。 当设置为 Hide 时，不可见的图表将被隐藏，以简化 DOM 并加快浏览器的速度。 设置为消去，降低浏览器的内存需求。 将其设置为 隐藏 以在页面滚动时更快地恢复图表。
如何处理页面卷动？
当设置为 同步 时，将在滚动后立即检查图表的可见性。 在速度较慢的计算机上，这可能会影响页面滚动的平滑度。 要在滚动结束时更新页面，请将其设置为 Async。 将其设置为 Sync 以在滚动时立即更新图表。 将其设置为 非同步，以便在较慢的计算机上更流畅地滚动页面。
采用何种图表更新模式？
当设置为并行时，可见图表并行刷新（所有查询并行发送到netdata服务器）并异步呈现。 当设置为 sequential 时，图表会一个接一个地刷新。 如果您的浏览器可以处理它（大多数现代浏览器都可以），请将其设置为并行，如果您在较旧/较慢的计算机上工作，则将其设置为顺序。
重新整理时是否需要同步所有图表？
当设置为同步时，dashboard将尝试重新同步所有图表，以便它们同时刷新。 当设置为尽力而为时，每个图表可能会刷新，与其他图表有一点时差。 通常，仪表板开始并行刷新它们，但根据您的计算机速度和网络延迟，图表开始有轻微的时间差异。 将此设置为 同步 将尝试在每次更新时重新同步图表。 将其设置为尽力而为可能会降低浏览器和网络的压力。
是否悬停在图表上时同步更新所有图表？
启用后，一个图表上的选择将自动选择所有其他可见图表上的相同时间，并且所有可见图表的图例将更新以显示所选值。 禁用时，只会选择引起用户注意的图表。 启用它以获得更好的数据洞察力。 如果您使用的计算机速度非常慢，实际上无法执行此操作，请禁用它。
是否同步移动与缩放所有图表？
启用后，图表上的平移和缩放操作将复制到所有图表（即使是当前不可见的图表 - 当然只有当它们变得可见时）。 禁用时，平移和缩放操作不会传播到其他图表。
使用那一种布景主题？
Netdata自带两个主题：简单（默认）和浅色。
切换此选项将重新加载仪表板。
您需要说明吗？
Netdata 可以在某些方面显示一些帮助，以帮助您使用仪表板。 如果所有这些气球都打扰您，请使用此开关禁用它们。
切换此选项将重新加载仪表板。
是否启用在移动与缩放时同时填补数据？
当设置为 Pad 时，图表将在可见区域之前和之后填充更多数据，从而给人一种整个数据库已加载的印象。 这种填充只会在图表上第一次平移或缩放操作之后发生（最初所有图表只有可见数据）。 当设置为不需入时，即使在第一次平移和缩放操作之后，也只会从netdata服务器传输可见数据。
是否在图表上启用贝兹曲线？
当设置为支持它的图表库时，将绘制平滑的曲线而不是简单的直线来连接点。
请记住 dygraphs，netdata 仪表板中的主要图表库，只能平滑折线图。 它不能平滑区域或堆积图。 当设置为 Rough 时，此设置可以降低浏览器消耗的 CPU 资源。
随着图表的更新，这些设置会逐渐应用。 要强制执行这些操作，请立即刷新仪表板。
是否启用自动缩放单位选择？
当设置为缩放单位时，显示的值将被动态缩放（例如1000 KB将显示为1兆位）。 Netdata 可以自动缩放这些原始单位：kilobits/s、kilobytes/s、KB/s、 KB, MB, and GB. 当设置为固定单位时，所有值将使用由netdata服务器维护的原始单位呈现。
使用何种温度单位？
设置仪表板的温度单位。
是否将秒转换为时间？
设置为时间时，显示秒的图表将显示DD:HH:MM:SS。 当设置为 秒 时，将显示原始秒数。
是否显示浏览者本地时间或是伺服器时间？
当设置为本地时间时，图表将使用您浏览器的本地时间。 当设置为 Server Time 时，图表将使用服务器时间。
将其设置为 本地时间 以获得一个通用的时间参考，无论服务器位于何处以及使用哪个时区（您的所有仪表板都将报告您的本地时间）。 当您需要将netdata图表与服务器日志文件进行比较时，将其设置为服务器时间。
